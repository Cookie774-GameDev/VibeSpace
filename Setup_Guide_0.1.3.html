1. Install

Jarvis ships as a single Tauri desktop app. The Windows installer is a standard MSI; macOS and Linux builds are produced from the same Rust + Vite project but aren't covered here.

  1. Download Jarvis_0.1.3_x64_en-US.msi from the release you were sent (or built locally from app/src-tauri/target/release/bundle/msi/).
  2. Double-click the MSI. Windows SmartScreen may prompt — choose More info → Run anyway for unsigned dev builds.
  3. The installer drops the app under %LocalAppData%\Programs\Jarvis by default and adds a Start menu shortcut.
  4. Launch it. The first window opens to onboarding.
Local-first by default. Your chats, agents, custom tools, skills, and memory live in a Dexie IndexedDB on this device. Cloud sync is opt-in (Starter tier and above) and never moves data without you toggling it on.

2. First run

Onboarding asks for a display name, then drops you on the chat canvas. The seeded Jarvis agent uses the local mock provider until you wire a real one — answers will be placeholder text. That's the next step.

3. Add your first provider

The fastest free path is Google's Gemini API. The free tier covers Gemini 2.5 Flash Lite (the new default model in 0.1.3) at zero cost with no card on file.

  1. Open aistudio.google.com/apikey and sign in with a Google account.
  2. Click Create API key. Copy the AIza… string.
  3. In Jarvis, press Mod+, to open Settings (Mod = Ctrl on Windows, Cmd on macOS).
  4. Click Providers in the left rail. Find Google in the list, paste the key, click Save.
  5. Close Settings. Type "hello" in the composer and press Mod+Enter.

Jarvis replies. You're done with setup. Every other provider — Groq (free Llama 3.3), Anthropic, OpenAI, OpenRouter, Together — works the same way: paste a key, save, chat.

Why Gemini Flash Lite? 0.1.3 swaps the default to the truly-free Google quota. Earlier builds picked Flash, which Google started metering on the free plan. New installs now never accidentally bump into a paid budget.

4. The action system

Jarvis 0.1.3 introduces Approve/Cancel actions. When Jarvis decides the right answer is to do something — open the Terminals swarm, launch Claude Code in a new pane, jump to a Settings tab, start an eye break — it emits a fenced action block in chat. The block renders as an inline card with the action's name, a one-line rationale, the parameters it would use, and two buttons: Approve and Cancel. Nothing runs until you click Approve.

Status flows pending → running → success / error / cancelled. The card colours track the state: copper for pending, amber spinner for running, sage for done, terracotta for error, muted for cancelled. The result is fed back into the chat thread so Jarvis can react on its next turn.

Built-in actions you can ask for

  • Open the Terminals swarm: "open the swarm" or "drop in Builder, Scout, Reviewer, Jarvis"
  • Run Claude Code in a new pane: "open Claude in this project"
  • Run OpenCode: "open opencode"
  • Run any shell command in a new pane: "run npm run dev"
  • Switch settings tabs: "open the providers tab" / "show the plans tab"
  • Toggle theme: "switch to light mode"
  • Start a 20-20-20 eye break: "I need an eye break"
  • Open a URL: "open the Groq dashboard"
  • Start a new chat / branch this conversation

Twenty-four actions ship in 0.1.3 across eight categories. The full list lives in lib/actions/registry.ts and shows up when you open the actions palette.

5. Custom tools

A custom tool is a saved preset of any built-in action: a friendly name, optional emoji, description, and the params you want frozen. Save it once; Jarvis can propose it from chat, and you can fire it from the actions palette.

  1. Open Jarvis. Click Tools in the left nav (Wrench icon) or say "open tools".
  2. Click New tool or pick a quick-start template.
  3. Pick a base action (e.g. terminal.run), fill in preset params (e.g. command: npm run dev, label: dev, cwd: C:\proj), give it a name like "Run my dev server", and save.
  4. The tool now appears under custom.run-my-dev-server. Try it: open the actions palette and search for it.
Local for now. Custom tools persist in browser storage on this device. Cloud sync for tools is "Available soon" — until then, use Export / Import (top of the Tools page) to move them between machines.

6. Actions palette

Press Mod+Shift+A. The actions palette opens — substring search, grouped by category, your most recent actions pinned at the top, plus every custom tool you've saved. Click an action: if it has required params, an inline form appears; otherwise it runs immediately. Esc dismisses.

The palette is sister to the existing two: Mod+K (general command palette — chats, nav, settings) and Mod+Shift+L (launcher tiles for URLs and pinned items). The actions palette is for "do something Jarvis can do".

7. 20-20-20 break

Every 20 minutes of screen time, look at something 20 feet away for 20 seconds. The new wellness break overlay enforces the third number: a full-screen takeover with a breathing copper orb and a serene countdown. Esc dismisses if you're mid-sentence.

To start one:

  • Ask Jarvis: "give me an eye break"
  • Open the actions palette and search "eye break"
  • Save a custom tool that wraps wellness.eyeBreak and pin it

9. Auto-start at 8 AM

To have Jarvis open every morning at 8 AM on Windows, register a Scheduled Task. Run this once in PowerShell (no admin required for the user-scope task): 

$exe = "$env:LocalAppData\Programs\Jarvis\Jarvis.exe"
$action = New-ScheduledTaskAction -Execute $exe
$trigger = New-ScheduledTaskTrigger -Daily -At 8:00am
Register-ScheduledTask `
  -TaskName 'JarvisMorningOpen' `
  -Action $action `
  -Trigger $trigger `
  -Description 'Open Jarvis daily at 8 AM'

To unregister it later: 

Unregister-ScheduledTask -TaskName 'JarvisMorningOpen' -Confirm:$false

If you installed Jarvis to a non-default location (or are running a locally-built debug binary), point $exe at the actual executable.

10. Hotkeys

Mod = Ctrl on Windows / Linux, Cmd on macOS.

  • Mod+K — Command palette (chats, nav, settings)
  • Mod+J — Jarvis Assistant bar (natural-language commands)
  • Mod+Shift+A — Actions palette (new)
  • Mod+Shift+L — Launcher tiles
  • Mod+Enter — Send message
  • Mod+Shift+Enter — Broadcast to council agents
  • Mod+Space — Push-to-talk voice
  • Mod+Shift+M — Voice-to-text in composer
  • Mod+Shift+F — Toggle fullscreen workspace
  • Mod+Shift+. — Toggle ambient mode
  • Mod+Shift+S — Open schedule
  • Mod+Shift+T — Toggle todo drawer
  • Mod+B — Toggle nav pane
  • Mod+\ — Toggle inspector pane
  • Mod+, — Settings
  • Esc — Close modal / exit council / dismiss wellness break

11. Troubleshooting

The seeded Jarvis agent answers with placeholder text.

No real provider key set yet. Paste a Gemini or Groq key under Settings → Providers and the agent will route through it.

Approve button on an action card is disabled.

The action id wasn't registered. Either the AI hallucinated the name or you deleted a custom tool between when Jarvis proposed it and when you tried to approve. Cancel the card and ask Jarvis to retry.

Mod+Shift+A doesn't open the palette.

Make sure no input field has focus first — the palette uses the standard "suppress hotkey while typing" rule. Click on empty canvas or press Esc first, then try again.

Terminals swarm doesn't appear after I approved it.

Fixed in 0.1.3 — earlier builds raced the lazy chunk on cold-load. If you see this on 0.1.3, please file an issue with the steps. The swarm now drains from a queue on page mount so it works whether the Terminals route was open already or not.

The wellness break never auto-ends.

The countdown uses requestAnimationFrame — if the OS throttles the window in the background, the timer pauses with it. Bring the window forward; the countdown resumes from the right second. Esc always works to skip.

How do I export everything?

Custom tools: Tools page → Export button (top right). Chats & memory: Settings → Account → Export local data. Both produce JSON files you can store anywhere.
